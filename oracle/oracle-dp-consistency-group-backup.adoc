= 一致性组备份
:allow-uri-read: 


[role="lead"]
为了获得最简单的备份，只需将整个 Oracle 数据库放在单个 ONTAP 卷、LUN 或 NVMe 命名空间中。

使用单个快照保护数据文件、存档日志、重做日志和控制文件是一种有效的备份、还原和复制方法。但是，RPO 仅限于备份本身的时间点。它适用于 1 小时或更长时间的 RPO。如果数据库跨 AFF 卷或 ASA 存储单元，则需要使用前面讨论的其中一种工具进行一致性组。

例如，整个数据库可以位于具有以下快照计划的单个 AFF 卷中：

* 72个每小时快照
* 30个夜间快照
* 12个月快照


此操作可在前 72 小时的滚动期间提供一小时的 RPO，并额外提供每晚和每月备份。您可以在单个卷或一组一致性组快照中包含多个数据库或应用程序文件，以便在更大的环境中执行一致的备份。
